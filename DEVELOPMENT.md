# SimpleGitClient — Developer Guide

A native macOS **GitKraken clone** built in Swift / SwiftUI. This document is the handoff
notes for the next developer (human or AI). Read it before changing anything.

---

## 1. What this is

- A desktop Git GUI: commit graph, staging/diff, branches, push/pull/fetch/clone/stash, `.gitignore`
  helper, credential management (incl. Overleaf push tokens), multi-tab with session restore.
- Dark theme, modeled on `../gitkraken-clone-ui-spec.md`. Intentionally **no** marketing / AI chrome
  (no Upgrade, no notifications, no "Agents", no bottom status bar, no Terminal panel).
- Single Swift Package, single module named `SimpleGitClient`. (Renamed from "GitPilot" on 2026-06-30.)
- The outer `GitClient/` directory is a workspace, **not** a git repo. The project lives in
  `GitClient/SimpleGitClient/`.

## 2. Tech stack & architecture

- **Swift 5.10 tools / Swift 5 language mode**, SwiftUI, macOS 14+. `@Observable` (Observation
  framework) for all view models — **not** ObservableObject/Combine.
- **MVVM-ish**:
  - `Models/` — value types (`GitCommit`, `GitBranch`, `GitFileStatus`, `DiffModels`, `GitGraph`,
    `FileTree`, `Credential`).
  - `ViewModels/` — `AppViewModel` (tabs, clone, auth, recents), `RepoViewModel` (per-repo state),
    `GraphViewModel`, `DiffViewModel`, `AuthViewModel`. All `@Observable` classes.
  - `Services/` — `GitService` (runs `/usr/bin/git` via `Process`), `KeychainService`,
    `FileWatcherService` (FSEvents), `ToastCenter`, `ActivityCenter`.
  - `Views/` — SwiftUI views grouped by area (TitleBar, Toolbar, Sidebar, Graph, Diff, Staging,
    Landing, Clone, Auth, Branch, Window, Toast).
  - `Theme/` — `Theme.swift` (colors/sizes/fonts, all from the spec) and `Styles.swift`
    (reusable `ButtonStyle`s + `SegmentButton` + `DropdownSelector`).
- **Git is shelled out**, no libgit2. Every command goes through `GitService.execute(_:at:)`.
- Entry point: `Sources/SimpleGitClientApp.swift` (`@main struct SimpleGitClientApp`). It injects
  `AppViewModel`, `ToastCenter.shared`, `ActivityCenter.shared` into the environment and applies
  `.windowStyle(.hiddenTitleBar)`.

## 3. Build / run / package

From `GitClient/SimpleGitClient/`:

```bash
swift build                          # typecheck only
scripts/build_app.sh [debug|release] # → dist/SimpleGitClient.app  (default: release)
open ./dist/SimpleGitClient.app
```

- A **bare SPM executable is not enough** to be a real macOS app — you must package the `.app`
  bundle (`scripts/build_app.sh` writes Info.plist, copies the icon, ad-hoc codesigns).
- The app icon is generated by `scripts/make_icon.sh` → `scripts/GenerateIcon.swift` (CoreGraphics,
  draws a teal line-art commit graph) → `Resources/AppIcon.icns`. `build_app.sh` auto-generates it
  if missing.
- **Test hook**: launch with `--open <repo-path>` (or env `SIMPLEGITCLIENT_OPEN`) to auto-open a repo
  and skip session restore. Used for scripted/visual testing.
- For computer-use screenshot verification, request screen access by **bundle id**
  `com.simplegitclient.app` (the display name does not resolve).

## 4. CRITICAL gotchas (do not regress these — each was a real bug)

1. **Process pipe deadlock.** `GitService.execute`/`executeWithStdin` MUST drain stdout AND stderr
   **concurrently** (DispatchGroup on a concurrent queue) while the process runs. Reading in
   `terminationHandler` after exit deadlocks on output > ~64KB (e.g. `git status --untracked-files=all`
   on a folder with thousands of untracked files → infinite spinner + empty file list). Keep the
   concurrent-drain implementation.

2. **`@Observable` singletons + overlays don't auto-render.** `ToastCenter`/`ActivityCenter` are
   `@Observable` singletons mutated outside the normal view event flow (from `Task { @MainActor }`).
   SwiftUI did **not** re-render the bottom overlays (`ToastOverlay`/`ActivityOverlay`) on their
   changes. Fix: `MainLayout` holds them as `@State private var toastCenter = ToastCenter.shared` /
   `activityCenter` and **reads their counts in `body`** (`let _ = toastCenter.toasts.count`) so
   MainLayout re-renders and rebuilds the overlays. If you refactor toasts/progress, preserve this
   observation anchor or the overlays will silently stop updating.

3. **Commit graph needs `git log --topo-order`.** Parents must come after children or the lane
   algorithm in `GitGraph.swift` breaks. Also: in `GraphLayoutCalculator`, iterate `before.indices`
   (a snapshot), never `lanes.indices`, because `firstFreeLane()` can grow `lanes` mid-loop
   (out-of-range crash otherwise).

4. **Valid `ClosedRange` for column dividers.** In `CommitGraphView`, the graph column's drag range
   is `graphMin...graphMax` where `graphMax = max(440, graphMin + 80)`. For repos with many lanes
   `graphMin` can exceed a fixed upper bound → `lowerBound > upperBound` → `ClosedRange` precondition
   crash. Never hardcode the upper bound.

5. **Ref decorations.** `parseRefs` skips `HEAD` and any `*/HEAD` (e.g. `origin/HEAD`) — otherwise a
   commit shows a duplicate "cloud" pill next to `origin/main`.

6. **Untracked files.** `git status` uses `--untracked-files=all` to list individual files (matches
   the spec / GitKraken), so the staging list uses `LazyVStack` to stay smooth with thousands of rows.

## 5. Auth (push & clone) — Overleaf etc.

- `GitService.execute` runs with `GIT_TERMINAL_PROMPT=0`. Auth failures are detected by matching
  stderr for `authentication` / `could not read username` / **`could not read password`** /
  `invalid username or password` / `403` → `GitError.authenticationRequired`. (Overleaf embeds
  `git@` in the URL, so it only lacks the *password* — matching "Username" alone is not enough.)
- On a push/clone auth failure, `AuthSheet` appears (shared via `AppViewModel.AuthContext.{push,
  clone}`). It is **token-first**: just a Token field (Overleaf needs no username; username defaults
  to `git`, hidden under "Advanced"). On submit it stores the token via `git credential approve` +
  osxkeychain (and optionally the app Keychain), then **retries the operation automatically**. The
  ⚙️ gear opens `CredentialsSheet` to pre-add a token for any host.

## 6. Implemented features (high level)

- Multi-tab with **session restore** (open repo paths persisted to UserDefaults
  `SimpleGitClient.openTabs` + `.activeTabPath`, restored in `AppViewModel.bootstrap()`). Note:
  `newTab()` must NOT persist (a path-less New Tab would overwrite the saved list).
- New Tab **landing page** (Open / Clone / Recent). **Clone** dialog (URL or GitHub, with destination,
  Browse, and a folder **Name** field). **Create branch** sheet.
- **Commit graph**: 3 resizable columns (`BRANCH/TAG | GRAPH | COMMIT MESSAGE`) with draggable
  dividers (`ColumnDivider`), WIP row, avatar-in-node, colored lanes, merge nodes, gap-free lines.
- **Diff view**: file header (status, UTF-8, Stage/Unstage), toolbar (Edit, File/Diff segment, hunk
  nav, whitespace/wrap), synthesized diff for untracked files, **File View** (works for working tree
  AND commit files via `git show <hash>:<path>`).
- **Staging**: collapsible Unstaged/Staged, Path/Tree modes, hover Stage/Unstage, discard, panel
  header with discard-all + "N file changes on <branch>". Right-click a file → **.gitignore** menu
  (ignore this file / folder / `*.ext`) + Discard.
- **Commit composer**: summary + char counter (72 hint), description, **Commit options** (sign-off
  `-s`, allow-empty — both functional), state-driven commit button.
- **Toasts** (bottom-left, solid green/red/blue, high-emphasis) for operation results;
  **progress indicator** (bottom-left, **yellow**, spinner + label) during pull/push/fetch/clone/etc.
- Hover/press feedback on buttons (`ToolbarButtonStyle`, `PrimaryButtonStyle`, `OutlineButtonStyle`,
  `SegmentButton`), traffic lights inline with the first tab (`WindowAccessor` + `.ignoresSafeArea(.top)`).

## 7. Conventions

- Match the surrounding code style: Chinese `///` doc comments are common; keep them.
- All colors/sizes come from `Theme`. Don't hardcode hex in views — add a `Theme` token.
- Reuse the `ButtonStyle`s in `Styles.swift` for new buttons so hover/press feedback is consistent.
- `RepoViewModel` git-action methods are `@MainActor`, wrap slow ops in
  `ActivityCenter.shared.track("…") { … }`, and emit a `ToastCenter.shared.show(…)` on success;
  errors surface as red toasts.

## 8. Known limitations / good next tasks

- Column widths and panel widths are **not persisted** across launches (per-session only).
- No conflict-resolution UI; merge/rebase conflicts only surface as error toasts.
- Undo/Redo and toolbar Search are placeholders (disabled / no-op).
- "GitHub.com" clone source is just a URL helper — no OAuth repo browser.
- Diff rendering is SwiftUI `LazyVStack` (fine for normal files; very large files could use NSTextView).
- The graph can get very wide for repos with many concurrent branches (column auto-sizes to lanes).
- The project folder itself has no `.gitignore`; if you `git init` it, `.build/` and `dist/` show as
  thousands of untracked files (the deadlock fix handles it, but add a `.gitignore` for sanity).

## 9. Testing discipline

- **Never use the user's real repositories as test fixtures.** Create disposable repos under the
  session scratchpad with `git init` + synthetic commits (include a branch + merge + untracked files
  to exercise the graph and staging). Clean them up afterward.
- To reproduce auth flows safely, point a throwaway repo at a fake host and **Cancel** before
  submitting — never store a fake token for a host the user actually relies on (overleaf/github).
